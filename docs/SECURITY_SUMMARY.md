## 🔐 ШИФРОВАНИЕ ДАННЫХ

### Уровень 1: Клиент (Flutter App)

```
┌─────────────────────────────────────────┐
│  Flutter Secure Storage                 │
│  ├─ Ed25519 Private Key (AES-256)      │
│  └─ X25519 Session Key (AES-256-GCM)   │
└─────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────┐
│  E2EE Шифрование                        │
│  ├─ AES-256-GCM (сообщения)             │
│  ├─ X25519 (обмен ключами)              │
│  └─ Ed25519 (подписи)                   │
└─────────────────────────────────────────┘
```

**Защита:**
- ✅ Приватные ключи в hardware-backed хранилище
- ✅ Сообщения шифруются ПЕРЕД отправкой
- ✅ Cloudflare получает только ciphertext

### Уровень 2: Передача (Network)

```
Клиент ──[TLS 1.3 / HTTPS]──> Cloudflare Edge ──[Internal]──> D1 Database
```

**Защита:**
- ✅ TLS 1.3 шифрование трафика
- ✅ Сертификаты Cloudflare
- ✅ HSTS (принудительный HTTPS)

---

### Уровень 3: Cloudflare D1

```
┌─────────────────────────────────────────┐
│  Cloudflare D1 Database                 │
│  ├─ encrypted_text: AES-256-GCM        │
│  ├─ nonce: 96-bit unique               │
│  └─ signature: Ed25519 (512-bit)       │
```

**Защита:**
- ✅ Данные зашифрованы НА КЛИЕНТЕ

- ✅ Vault триггеры блокируют изменения

```bash

"U2FsdGVkX19abc123def456ghi789..."  ❌ НЕЧИТАЕМО
```

**Вывод:** ❌ **НЕВОЗМОЖНО** без сессионного ключа

```

{
  "encrypted_text": "U2FsdGVkX19...",
  "nonce": "abc123"

```

**Вывод:** ❌ **НЕВОЗМОЖНО** — TLS 1.3 + E2EE

### Сценарий 3: Физический доступ к телефону
```
/data/data/com.liberty_reach/
├─ shared_prefs/
│  └─ flutter_secure_storage.xml  (AES-256 encrypted)
└─ databases/
   └─ messages.db  (ciphertext only)
```

**Вывод:** ⚠️ **ОЧЕНЬ СЛОЖНО** — требуется:
- Root доступ
- TEE exploitation
- Брутфорс PIN/биометрии

## ⚠️ СЛАБЫЕ МЕСТА

1. **Метаданные可见**
   - Cloudflare видит: кто → кому → когда
   - Не видит: ЧТО (содержимое)

2. **Доверие к Flutter Secure Storage**
   - Закрытый исходный код плагина
   - Зависимость от команды разработчиков

3. **Физический доступ + Root**
   - TTEE extraction возможен
   - Требуется дорогое оборудование

4. **Социальная инженерия**
   - Фишинг для получения доступа
   - Не техническая уязвимость
